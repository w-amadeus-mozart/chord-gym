# Repository instructions

## Commit and push policy

When a task results in a working, verified change (build passes, relevant regression
tests pass), commit it and push to the current branch without asking for confirmation
first. This is a standing authorization for this repository — do not ask "should I
commit?" or "should I push?" for routine work here.

This does not lift other safety defaults: still scope commits to files related to the
task at hand, write clear commit messages, and never force-push, rewrite history, or
push to a branch other than the one currently checked out without being explicitly
asked.
